## 📋 프로젝트 이해

### /요약
- **사용법**: `/요약`
- **기능**: PROJECT_SUMMARY.md 읽고 3줄로 설명
- **예시**: `/요약`

---

## 🔧 기능 수정

### /수정
- **사용법**: `/수정 [파일:라인] [문제]` 또는 `/수정 [문제 설명]`
- **기능**: 버그나 기존 기능 개선
- **예시**:
  - `/수정 ExportService.cs:20 null 체크 추가`
  - `/수정 ExportService에서 null 참조 버그 수정`

---

## ✨ 새로운 기능

### /신규
- **사용법**: `/신규 [기능 설명]`
- **기능**: 새로운 기능 추가 (유형B)
- **예시**:
  - `/신규 CSV 데이터 추출 기능 추가`
  - `/신규 LogicProcessor 기능 추가`

## 🔄 작업 재개

### /재개
- **사용법**: `/재개 WIP-YYYYMMDD-NN`
- **기능**: 이전 WIP에서 작업 재개
- **예시**: `/재개 WIP-20250208-001`

### /CONTINUE
- **사용법**: `/CONTINUE WIP-YYYYMMDD-NN`
- **기능**: 이전 WIP에서 작업 재개 (위와 동일)
- **예시**: `/CONTINUE WIP-20250208-001`

## ✅ 작업 완료/취소

### /완료
- **사용법**: `/완료 WIP-YYYYMMDD-NN`
- **기능**: WIP 작업 완료 처리

### /취소

- **기능**: WIP 작업 취소 처리
- **예시**: `/취소 WIP-20250208-001 우선순위 조정으로 인해`

### /상태
- **사용법**: `/상태 WIP-YYYYMMDD-NN` 또는 `/상태 전체`

  - `/상태 WIP-20250208-001`
  - `/상태 전체`

## 📤 내보내기

### /긴급버그
- **사용법**: `/긴급버그 [파일:라인] [오류 메시지]` 또는 `/긴급버그 [문제 설명]`

## 💾 커밋 및 푸시 (COMMIT_RULES.md 따름)

### /커밋
- **사용법**: `/커밋 [메시지]` 또는 `/커밋` (메시지 없으면 자동 생성)
- **기능**: 로컬 변경 사항 커밋
- **메시지 자동 생성**: 메시지가 없으면 git status와 git diff로 변경 사항 분석 후 [태그] 요약 형식으로 자동 생성
- **태그 형식**: [feat], [fix], [refactor], [docs], [style], [test], [chore], [perf], [ci], [revert]
- **WorkID 포함**: 해당 WIP가 있는 경우 본문에 WorkID 포함
- **예시**:
  - `/커밋 [feat] CSV 데이터 추출 기능 추가` (사용자 지정 메시지)
  - `/커밋` (변경 사항 분석 후 자동 메시지 생성)

### /푸시
- **사용법**: `/푸시 [메시지]` 또는 `/푸시` (메시지 없으면 자동 생성)
- **기능**: 원격 저장소로 푸시
- **메시지 자동 생성**: 메시지가 없으면 git log로 최근 커밋 분석 후 메시지 자동 생성
- **예시**:
  - `/푸시 CSV 기능 추가 및 버그 수정` (사용자 지정 메시지)
  - `/푸시` (최근 커밋 분석 후 자동 메시지 생성)

### /전송
- **사용법**: `/전송`
- **기능**: 스테이징 → 커밋 → 푸시 한번에 진행
- **변경 사항 분류**: 관련 내용별로 그룹화하여 별도 커밋 (하나의 논리적인 변경 단위)
- **메시지 자동 생성**: 각 그룹별로 [태그] 요약 형식으로 자동 생성
- **예시**:
  ```
  /전송
  → 분석 결과:
     1. ExportService.cs, ExportViewModel.cs (CSV 기능)
     2. ExportServiceTests.cs (테스트)
     3. README.md (문서)
  → 커밋 1: [feat] CSV 데이터 추출 기능 추가
  → 커밋 2: [test] CSV 추출 기능 테스트 추가
  → 커밋 3: [docs] CSV 기능 관련 문서 업데이트
  → 푸시 완료
  ```

## 💡 팁

- `/명령어`를 입력하면 이 목록을 다시 볼 수 있습니다.
- 각 명령어는 AGENTS.md의 에이전트 워크플로우를 자동화합니다.
- WorkID는 자동 생성되며 형식은 `WIP-YYYYMMDD-NN`입니다.
- 커밋 관련 명령어(`커밋`, `푸시`, `전송`)는 COMMIT_RULES.md를 따릅니다.
