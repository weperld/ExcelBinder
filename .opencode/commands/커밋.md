---
description: 커밋 - 로컬 변경 사항 커밋
---

다음 메시지로 커밋을 진행해주세요:

[커밋 메시지]: $ARGUMENTS

**작업 절차**:
1. git status로 변경 사항 확인
2. git diff로 변경 내용 검토
3. 변경 사항을 적절히 분류 (add)
4. 메시지 처리:
   - 메시지가 있는 경우: 지정된 메시지로 커밋
   - 메시지가 없는 경우: 변경 사항을 분석하여 [태그] 요약 형식으로 메시지 자동 생성
5. git status로 커밋 확인

**메시지 자동 생성 규칙** (메시지가 없는 경우):
1. 변경 파일 타입 분석 (COMMIT_RULES.md 따름):
   - 새로운 기능 추가 → [feat]
   - 버그 수정 → [fix]
   - 코드 리팩터링 (기능 동일) → [refactor]
   - 문서 추가/수정 (코드 X) → [docs]
   - 코드 포맷팅, 스타일 → [style]
   - 테스트 코드 추가/수정 → [test]
   - 빌드 시스템, 의존성, 도구 → [chore]
   - 성능 개선 → [perf]
   - CI/CD 설정 변경 → [ci]
   - 이전 커밋 되돌리기 → [revert]
2. 변경 내용 요약:
   - 어떤 파일들이 변경되었는지
   - 주요 변경 사항이 무엇인지
   - 한글로 간결하게 작성 (50자 이내 권장)
   - 대문자로 시작, 마침표 금지
3. 형식: [태그] 요약
4. WorkID 포함 (해당 WIP가 있는 경우):
   - 본문에 `WorkID: WIP-YYYYMMDD-NN` 형식으로 포함
   - 빈 줄로 제목과 분리

**커밋 메시지 형식**:
- [태그] 요약
- 태그: [추가], [수정], [버그수정], [리팩토링], [문서], [테스트] 등
- 요약은 한글로 간결하게 작성

**요구사항**:
- 절대 규칙 위반 코드는 커밋하지 않음
- 불필요한 파일(.env, credentials.json 등)은 제외
- 커밋 메시지는 명확하고 구체적으로

**예시**:

사용자 지정 메시지:
```
/커밋 [feat] CSV 데이터 추출 기능 추가
/커밋 [fix] ExportService null 참조 수정
```

메시지 없는 경우 (자동 생성):
```
/커밋
→ 분석 결과: ExportService.cs 추가, ExportViewModel.cs 수정 (새로운 기능)
→ 생성 메시지: [feat] CSV 데이터 추출 기능 추가

/커밋
→ 분석 결과: ExportService.cs 수정 (null 체크 추가)
→ 생성 메시지: [fix] ExportService null 참조 수정

/커밋 (WIP와 연결된 경우)
→ 분석 결과: ExportService.cs 수정 (비동기 패턴 개선)
→ 생성 메시지:
[refactor] ExportService 비동기 처리 개선

WorkID: WIP-20250208-001
```
