---
description: 푸시 - 원격 저장소로 푸시
---

다음 메시지로 푸시를 진행해주세요:

[푸시 메시지]: $ARGUMENTS

**작업 절차**:
1. git status로 상태 확인
2. 로컬 커밋이 있는지 확인
3. 메시지 처리:
   - 메시지가 있는 경우: 지정된 메시지로 푸시
   - 메시지가 없는 경우: git log로 최근 커밋 분석하여 메시지 자동 생성
4. 원격 저장소로 푸시
5. 푸시 성공 확인

**메시지 자동 생성 규칙** (메시지가 없는 경우):
1. git log로 최근 커밋 확인 (최근 1-3개)
2. 커밋 메시지 분석 (태그, 요약, 유형) → COMMIT_RULES.md 참조 (Line 25-36)
3. 푸시 메시지 생성:
   - 단일 커밋: 해당 커밋 메시지의 요약 부분 사용
   - 여러 커밋: 주요 변경 사항 종합하여 요약
   - 형식: 간결한 요약 (한글, 50자 이내 권장)

**푸시 메시지 형식**: 커밋과 유사한 형식 → COMMIT_RULES.md 참조 (Line 7-15)

**요구사항**:
- force push 금지 (main/master 브랜치)
- 커밋이 완료된 후에만 푸시
- 충돌 시 사용자에게 알림

**예시**:

사용자 지정 메시지:
```
/푸시 CSV 기능 추가 및 버그 수정
/푸시 [refactor] 아키텍처 개선
```

메시지 없는 경우 (자동 생성):
```
/푸시
→ 분석 결과: 최근 1개 커밋 "[feat] CSV 데이터 추출 기능 추가"
→ 생성 메시지: CSV 데이터 추출 기능 추가

/푸시
→ 분석 결과: 최근 2개 커밋 "[feat] CSV 데이터 추출 기능 추가", "[fix] ExportService null 참조 수정"
→ 생성 메시지: CSV 기능 추가 및 버그 수정
```
