## 작업 진행 내용

### 1단계: 확인 및 분석 (Confirm & Analyze)

#### 1.1. Plan 단계 결과물 검증
- [ ] 코디네이터로부터 받은 지시 문서에서 계획 요약 확인
- [ ] 영향 파일 목록 확인
- [ ] 위험 요소 및 대응책 확인
- [ ] 사용자 확인 상태 확인

#### 1.2. 기존 아키텍처 분석
- [ ] 기존 클래스 구조 확인
- [ ] 기존 패턴 확인 (MVVM, Async 등)
- [ ] 기존 의존성 확인
- [ ] 기존 성능 특성 확인

#### 1.3. 설계 요구사항 분석
- [ ] 새로운 컴포넌트/기능의 기술적 요구사항 파악
- [ ] 기존 시스템과의 통합 요구사항 파악
- [ ] 성능/안정성 요구사항 파악

#### 1.4. 더블체크 계획 (Gate-2 준비)
- [ ] 1차 검증 항목 정의: 아키텍처 완성도, 기술 규칙 준수
- [ ] 2차 검증 항목 정의: 설계 실현 가능성, 기술 규칙 준수
- [ ] 크로스체크 에이전트 지정: @developer

#### 1.5. 검증
- [ ] Plan 단계 결과물이 올바르게 전달되었는가?
- [ ] 기존 아키텍처가 파악되었는가?
- [ ] 설계 요구사항이 명확한가?
- [ ] 더블체크 계획이 준비되었는가?

---

### 2단계: 해야할 일에 대한 계획 수립 (Plan)

#### 2.1. 구체적인 아키텍처 설계
- [ ] 클래스/메서드 구조 정의
- [ ] 인터페이스 정의 (필요한 경우)
- [ ] 의존성 주입 구조 정의
- [ ] 데이터 흐름 정의

#### 2.2. 기술적 검증 및 문제점 식별
- [ ] 성능 고려사항 분석
- [ ] 스레드 안전성 분석
- [ ] 메모리 관리 분석
- [ ] 잠재적 문제점 식별

#### 2.3. 설계 리스크 평가 및 해결책 제안
- [ ] 설계 리스크 식별
- [ ] 각 리스크에 대한 해결책 제안
- [ ] 우선순위 정의

#### 2.4. Code 단계를 위한 상세 설계 문서 작성
- [ ] 클래스 다이어그램 작성
- [ ] 메서드 시그니처 정의
- [ ] 인터페이스 명세 작성
- [ ] 기술 가이드라인 작성

#### 2.5. 크로스체크 요청 계획 (Gate-2 준비)
- [ ] @developer에게 요청할 검증 항목 정의:
  1. 설계 실현 가능성 검증
  2. 기술 규칙 준수 검증
  3. 구현 난이도 검증
- [ ] 검증 기준 설정

#### 2.6. 검증
- [ ] 아키텍처 설계가 완전한가?
- [ ] 기술적 검증이 충분한가?
- [ ] 리스크 평가가 적절한가?
- [ ] 해결책이 현실적인가?
- [ ] 설계 문서가 충분히 상세한가?
- [ ] 크로스체크 요청 계획이 준비되었는가?

### 3단계: 해야할 일 진행 (Execute)

#### 3.1. 더블체크 1차 수행
- [ ] 아키텍처 완성도 확인
- [ ] 기술 규칙 준수 확인
- [ ] 성능/안정성 고려사항 확인
- 결과: 1차 통과 또는 수정 필요

#### 3.2. 크로스체크 요청 (@developer)
- [ ] @developer에게 크로스체크 요청 전송
- [ ] @developer로부터 피드백 수신
- 결과: 통과 또는 수정 요청

#### 3.3. 더블체크 2차 수행 (크로스체크 통과 후)
- [ ] 설계 실현 가능성 검증
- [ ] 기술 규칙 준수 검증
- [ ] 구현 난이도 검증
- 결과: 2차 통과 또는 수정 필요

#### 3.4. Gate-2 최종 통과
- [ ] 1차 더블체크 통과
- [ ] 크로스체크 통과
- [ ] 2차 더블체크 통과
- 통과: ✅

#### 3.5. 작업 순서
1. **최종 검증**: 위 검증 항목 확인
2. **결과 처리**:
   - 통과인 경우:
     1. 독립 WIP 완료 상태로 업데이트
     2. 지시자(코디네이터/사용자)에게 완료 보고
   - 수정 필요인 경우:
     1. 수정 내용 정리
     2. 재검증 요청
   - 취소인 경우:
     1. 취소 사유 기록
     2. 독립 WIP 취소 상태로 업데이트
3. **최종 보고**:
   - 완료: "✅ Design 단계 완료"
   - 수정 필요: "⚠️ 수정 필요: [내용]"
   - 취소: "❌ Design 단계 취소: [사유]"
