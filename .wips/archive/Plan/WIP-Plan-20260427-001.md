## 지시 정보

### 작업 대상 (Target)
Dashboard에 **Feature 그룹화** 기능 추가. Feature를 카테고리별로 분류하여 좌측 사이드바에서 선택 가능하게 함. 한 Feature가 여러 그룹에 속할 수 있는 다대다 매핑 모델.

### 변경 파일 (Files)
**신규 파일 (3개)**:
1. `ExcelBinder/Models/FeatureGroup.cs` — 그룹 모델
2. `ExcelBinder/Services/FeatureGroupService.cs` — `_groups.json` 입출력
3. (선택) `Views/Common/GroupContextMenu.xaml` — 재사용 가능한 ContextMenu (인라인으로 처리 가능)

**수정 파일 (3개)**:
4. `ExcelBinder/Models/ProjectConstants.cs` — 그룹 관련 상수
5. `ExcelBinder/ViewModels/MainViewModel.cs` — Groups 컬렉션, SelectedGroup, 필터링, 커맨드
6. `ExcelBinder/Views/DashboardView.xaml` — 좌측 사이드바 + ContextMenu

### 필수 조건 (Requirements)

**FEATURE_BACKLOG.md #3 정책 (12개 결정사항 사용자 합의 완료):**
1. 모델: 별도 `FeatureGroup` 클래스 (Id, Name, FeatureIds[])
2. 저장 위치: `Features/_groups.json` (Features 폴더 내부)
3. 다중 그룹 소속: 허용 (한 Feature가 여러 그룹에 동시 가능)
4. 정합성: 표시 시점 dangling ID 무시 (lazy GC), 그룹 편집 시 청소
5. UI: 좌측 사이드바 그룹 리스트 + 우측 Feature WrapPanel
6. 그룹 관리: 사이드바 우클릭 ContextMenu (추가/이름변경/삭제)
7. Feature↔그룹 매핑: Feature 우클릭 → 그룹 서브메뉴 체크박스 토글 (즉시 반영)
8. 그룹 ID: GUID
9. 그룹명 중복: 거부 (저장 시 검증, 사용자에게 메시지)
10. 사이드바 빈 공간 우클릭: "새 그룹 만들기" 노출 + 별도 "+" 버튼 병행
11. Feature 토글 UI: 서브메뉴 + 체크박스 (이미 속한 그룹 체크 표시, 클릭하면 제거)
12. "전체" 그룹 표시명: "전체"

**기술 제약:**
- "전체" 그룹은 가상 (저장소 미존재, 런타임 생성, 항상 첫 번째, 이름변경/삭제 불가)
- `FeatureDefinition` 모델 무변경 (Feature는 자신의 소속 그룹을 모름)
- 기존 Feature JSON 하위 호환 유지 (그룹화 도입 전 Feature는 "전체" 그룹에서만 보임)
- 그룹 미지정 Feature는 "전체"에서 항상 보임 (어느 커스텀 그룹에도 속하지 않을 수 있음)
- 절대 규칙 준수: 빈 catch 금지, 무조건 캐스팅 금지, dynamic 최소화
