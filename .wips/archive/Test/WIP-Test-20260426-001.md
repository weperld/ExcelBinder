### 3. ColumnName Fallback 보호 검증
TypeParser의 마지막 토큰(`parts.Last()`)이 ColumnName으로 잡히는 fallback 로직에서 다음 모든 케이스가 제외 처리됨:
- `BaseType` (라인 65)
- `ReferenceMarker` ("ref")
- `EnumMarker` ("enum")
- `RefType` 값
- `EnumType` 값

→ `"int:enum:Fire"` 처리 시 last="Fire" → EnumType="Fire"로 잡혔으므로 ColumnName fallback에서 제외됨. ✅

### 4. SchemaFieldViewModel 자동 초기화 검증 (정적 추론)
```csharp
// SelectedType setter
if (!SetProperty(ref _selectedType, value)) return;
OnPropertyChanged(nameof(CanInputEnum));
if (!CanInputEnum && !string.IsNullOrEmpty(_enumType))
{
    EnumType = string.Empty;   // 비정수형 전환 시 EnumType 클리어
}
```
시나리오:
- 사용자가 `int` → `MyEnum` 입력 → `string`으로 변경 → EnumType이 빈 문자열로 자동 초기화 ✅
- Save 시 `field.CanInputEnum && !string.IsNullOrWhiteSpace(field.EnumType)` 양쪽 조건 검사 → 비정수형이면 토큰 추가 안 됨 ✅

### 5. ExportService Binary/JSON enum 처리
- `WritePrimitiveOrEnum`: IsEnum이면 `writer.Write(value ?? "")` → BinaryWriter.Write(string)는 length-prefixed UTF-8 ✅
- `ParsePrimitiveOrEnum`: IsEnum이면 `value ?? ""` 그대로 반환 → JSON에 string으로 직렬화 ✅
- 빈 셀 → 빈 문자열 → 런타임 `Enum.Parse("")` 예외 (사용자 의도된 검증)

### 6. CodeGeneratorService 생성 코드 형태
```csharp
// 일반 int 필드
data.Hp = reader.ReadInt32();

// enum 필드 (int:enum:Element)
data.Element = System.Enum.Parse<Element>(reader.ReadString());

// List<int:enum:Effect>
// (템플릿이 ReadMethod 토큰을 List 루프에 적용)
data.Effects.Add(System.Enum.Parse<Effect>(reader.ReadString()));
```
→ `ConvertPrimitive`가 `info.EnumType` 반환, `GetReadMethod(TypeInfo)`가 `Enum.Parse` 인라인 ✅
