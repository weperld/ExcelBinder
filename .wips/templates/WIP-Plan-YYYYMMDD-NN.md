## 작업 진행 내용

### 1단계: 확인 및 분석 (Confirm & Analyze)

#### 1.1. 사용자 요구사항 검증
- [ ] 요구사항이 명확한가?
- [ ] 모호한 부분 식별
- [ ] 사용자 문의 필요 시 결정

#### 1.2. 기존 상태 확인
- [ ] 기존 파일 구조 확인
- [ ] 기존 기능과의 관련성 확인
- [ ] 충돌 가능성 확인

#### 1.3. 더블체크 계획 (Gate-1 준비)
- [ ] 1차 검증 항목 정의: 요구사항 명확성, 작업 범위 완전성
- [ ] 2차 검증 항목 정의: 기존 상태 파악, 위험 요소 식별
- [ ] 크로스체크 에이전트 지정: @architect

#### 1.4. 검증
- [ ] 요구사항이 명확한가?
- [ ] 작업 범위가 정의되어 있는가?
- [ ] 기존 상태가 파악되어 있는가?
- [ ] 더블체크 계획이 준비되었는가?

---

### 2단계: 해야할 일에 대한 계획 수립 (Plan)

#### 2.1. 유형 판단 (수정 A / 신규 B)
- [ ] 요구사항 키워드 분석
- [ ] 판단 점수 계산
- [ ] 최종 유형 결정

#### 2.2. 영향 파일 분석
- [ ] 필요한 파일 목록 작성
- [ ] 수정/생성 파일 구분
- [ ] 파일 간 의존성 확인

#### 2.3. 위험 요소 식별
- [ ] 호환성 위험 식별
- [ ] 데이터 무결성 위험 식별
- [ ] 성능 위험 식별

#### 2.4. 대응책 제안
- [ ] 각 위험 요소에 대한 대응책 수립
- [ ] 우선순위 정의

#### 2.5. 계획 수립
- [ ] 작업 순서 정의
- [ ] 예상 소요 시간 산정
- [ ] 마일스톤 정의

#### 2.6. 크로스체크 요청 계획 (Gate-1 준비)
- [ ] @architect에게 요청할 검증 항목 정의:
  1. 계획 명확성 검증
  2. 영향 파일 완전성 검증
  3. 위험 요소 및 대응책 적절성 검증
- [ ] 검증 기준 설정

#### 2.7. 검증
- [ ] 유형 판단이 올바른가?
- [ ] 영향 파일 목록이 완전한가?
- [ ] 위험 요소가 모두 식별되었는가?
- [ ] 대응책이 적절한가?
- [ ] 계획이 현실적인가?
- [ ] 크로스체크 요청 계획이 준비되었는가?

### 3단계: 해야할 일 진행 (Execute)

#### 3.1. 더블체크 1차 수행
- [ ] 요구사항 명확성 확인
- [ ] 작업 범위 완전성 확인
- [ ] 기존 상태 파악 확인
- [ ] 위험 요소 식별 확인
- 결과: 1차 통과 또는 수정 필요

#### 3.2. 크로스체크 요청 (@architect)
- [ ] @architect에게 크로스체크 요청 전송
- [ ] @architect로부터 피드백 수신
- 결과: 통과 또는 수정 요청

#### 3.3. 더블체크 2차 수행 (크로스체크 통과 후)
- [ ] 유형 판단 검증
- [ ] 영향 파일 완전성 검증
- [ ] 위험 요소 및 대응책 적절성 검증
- 결과: 2차 통과 또는 수정 필요

#### 3.4. Gate-1 최종 통과
- [ ] 1차 더블체크 통과
- [ ] 크로스체크 통과
- [ ] 2차 더블체크 통과
- 통과: ✅

#### 3.5. 작업 순서
1. **최종 검증**: 위 검증 항목 확인
2. **결과 처리**:
   - 통과인 경우:
     1. 독립 WIP 완료 상태로 업데이트
     2. 지시자(코디네이터/사용자)에게 완료 보고
   - 수정 필요인 경우:
     1. 수정 내용 정리
     2. 재검증 요청
   - 취소인 경우:
     1. 취소 사유 기록
     2. 독립 WIP 취소 상태로 업데이트
3. **최종 보고**:
   - 완료: "✅ Plan 단계 완료"
   - 수정 필요: "⚠️ 수정 필요: [내용]"
   - 취소: "❌ Plan 단계 취소: [사유]"
