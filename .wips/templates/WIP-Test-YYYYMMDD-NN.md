### 3단계: 해야할 일 진행 (Execute)

#### 3.1. 더블체크 1차 수행
- [ ] 테스트 커버리지 확인
- [ ] 버그 0개 확인
- [ ] 테스트 성공 기준 충족 확인
- 결과: 1차 통과 또는 수정 필요

#### 3.2. 크로스체크 요청 (@doc-manager)
- [ ] @doc-manager에게 크로스체크 요청 전송
- [ ] @doc-manager로부터 피드백 수신
- 결과: 통과 또는 수정 요청

#### 3.3. 더블체크 2차 수행 (크로스체크 통과 후)
- [ ] 테스트 결과 신뢰성 검증
- [ ] 버그 리포트 완전성 검증
- [ ] 테스트 결과 문서화 확인
- 결과: 2차 통과 또는 수정 필요

#### 3.4. Gate-4 최종 통과
- [ ] 1차 더블체크 통과
- [ ] 크로스체크 통과
- [ ] 2차 더블체크 통과
- 통과: ✅

#### 3.5. 작업 순서
1. **최종 검증**: 위 검증 항목 확인
2. **결과 처리**:
   - 통과인 경우:
     1. 독립 WIP 완료 상태로 업데이트
     2. 지시자(코디네이터/사용자)에게 완료 보고
   - 수정 필요인 경우:
     1. 수정 내용 정리
     2. 재검증 요청
   - 취소인 경우:
     1. 취소 사유 기록
     2. 독립 WIP 취소 상태로 업데이트
3. **최종 보고**:
   - 완료: "✅ Test 단계 완료"
   - 수정 필요: "⚠️ 수정 필요: [내용]"
   - 취소: "❌ Test 단계 취소: [사유]"
