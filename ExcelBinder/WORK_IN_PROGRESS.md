# 작업 추적 및 재매 시스템

> 작업 중단 시 이 파일을 사용하여 상태를 저장하고, 새 대화에서 재개합니다.

---

## 🔄 작업 재개 방법

### 새 대화에서 재매 시
```
1. 사용자: "재개: WIP-YYYYMMDD-NN"
   또는
   사용자: "CONTINUE: WIP-YYYYMMDD-NN"

2. 에이전트:
   - WORK_IN_PROGRESS.md 읽기
   - 해당 WorkID 상태 확인
   - 마지막 중단 시점 파악
   - "다음 작업" 확인
   - 즉시 재개

3. 재개 완료 후:
   - "🔄 WIP-YYYYMMDD-NN 작업이 재개되었습니다"
   - "현재 단계: [완료된 단계]"
   - "다음 작업: [다음 작업]"
```

## 🎯 WorkID 형식

### 추천 형식: **WIP-YYYYMMDD-NN**

**구조:**
- `WIP`: Work In Progress (고정 접두사)
- `YYYYMMDD`: 날짜 (예: 20250202)
- `NN`: 같은 날짜 내 순서 (01, 02, 03...)

**예시:**
```
WIP-20250202-001  # 2025년 2월 2일 첫 번째 작업
WIP-20250202-002  # 같은 날 두 번째 작업
WIP-20250203-001  # 다음 날 첫 번째 작업
```

**장점:**
- ✅ 날짜 기준으로 자동 정렬
- ✅ 충돌 가능성 최소화
- ✅ 히스토리 추적 용이
- ✅ 읽기 쉬움

## 🎯 에이전트 동작 가이드

### 작업 시작 시
1. 새로운 WorkID 생성: `WIP-YYYYMMDD-NN`
2. 활성 작업에 추가
3. 상세 정보 기록

### 진행 중 업데이트
1. 진행 상황에 타임스탬프 기록
2. 완료 단계 체크
3. 필요 시 사용자에게 진행 상황 보고

### 작업 완료 시
1. 완료 단계 모두 체크
2. 완료 정보 수집
3. 완료 작업으로 이동
4. 상세 정보 보존

### 작업 취소 시
1. 취소 사유 기록
2. 취소 작업으로 이동
3. 상세 정보 보존
